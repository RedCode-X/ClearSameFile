# 重复文件清理工具 — 需求文档

## 1. 项目概述

### 1.1 项目名称
**ClearSameFile** — Windows 重复文件查找与清理工具

### 1.2 项目目标
开发一款带图形界面的 Windows 桌面工具，帮助用户扫描磁盘中的重复文件，支持按文件哈希值精确比对，允许用户选择性删除重复文件以释放磁盘空间。

### 1.3 最终交付物
- 单个 `.exe` 可执行文件（通过 PyInstaller 打包）
- 免安装，双击即可运行

---

## 2. 技术栈

| 类别 | 选型 | 说明 |
|------|------|------|
| 语言 | Python 3.10+ | |
| GUI框架 | PySide6 / PyQt6 | 跨平台、原生控件、成熟稳定 |
| 打包工具 | PyInstaller | 将 Python 脚本打包为单个 EXE |
| 哈希算法 | MD5 + SHA256 | MD5 快速初筛，SHA256 二次确认 |
| 并发处理 | QThread / ThreadPoolExecutor | 扫描不阻塞 UI |

## 3. 功能需求

### 3.1 扫描管理

| 编号 | 功能 | 描述 |
|------|------|------|
| F-01 | 添加扫描目录 | 用户可选择一个或多个文件夹加入扫描列表 |
| F-02 | 移除扫描目录 | 用户可移除已添加的目录（单选/多选） |
| F-03 | 扫描范围过滤 | 支持设置最小/最大文件大小范围，过滤特定文件类型（如 .jpg, .mp4） |
| F-04 | 排除目录 | 支持排除特定子目录（如 node_modules、.git） |
| F-05 | 启动/停止扫描 | 点击开始扫描，支持中途取消，取消时保留已扫描的结果 |

### 3.2 重复检测

| 编号 | 功能 | 描述 |
|------|------|------|
| F-06 | 文件大小初筛 | 先按文件大小分组，大小不同的直接跳过 |
| F-07 | 部分哈希预筛 | 对大小相同的文件，读取前 4KB 算 MD5，快速排除内容不同的文件 |
| F-08 | 完整哈希确认 | 对通过预筛的文件，计算完整文件内容的 SHA256 以确认重复 |
| F-09 | 精确匹配 | 仅当文件大小和完整哈希值完全一致时，判定为重复文件 |

### 3.3 结果展示

| 编号 | 功能 | 描述 |
|------|------|------|
| F-10 | 重复分组列表 | 以分组形式展示重复文件，每组显示文件数量、总大小、可释放空间 |
| F-11 | 文件详情面板 | 选中某组后展示该组内所有文件的路径、大小、修改日期 |
| F-12 | 文件预览 | 对图片格式支持缩略图预览，双击可用系统默认程序打开文件 |
| F-13 | 排序与筛选 | 支持按文件大小、数量、路径等排序；支持搜索文件名 |
| F-14 | 统计信息 | 显示扫描文件总数、重复文件数、可释放总空间 |

### 3.4 选择与清理

| 编号 | 功能 | 描述 |
|------|------|------|
| F-15 | 自动选择策略 | 4 种全局策略：**智能保留**（分析路径+文件名关键词，自动识别原始文件）、保留最新、保留最旧、保留最短路径 |
| F-15.1 | 信任目录白名单 | 用户自定义信任目录列表，**信任目录内的所有文件必须保留**，其余文件全部标记为删除。策略仅在组内无信任文件时生效 |
| F-16 | 手动勾选 | 用户可逐组勾选要删除的文件，至少保留一个文件不被勾选 |
| F-17 | 全选/反选 | 支持按组或全局的全选/反选操作 |
| F-18 | 删除确认 | 执行删除前弹窗二次确认，显示将要删除的文件数量和总大小 |
| F-19 | 备份目录 | 删除的文件移入 `~/ClearSameFile_Backup/`，保留原始目录结构，附带 manifest.json 删除清单，支持从工具菜单一键恢复 |
| F-20 | 删除进度 | 删除操作显示进度条，完成后展示删除结果（成功/失败数量） |

#### 3.4.1 智能保留策略算法

「智能保留」综合**文件名**和**路径**两个维度打分，选最低分的文件保留（分数越低越可能是原始文件）。

**文件名扣分项**（命中 +1000 分）：

| 模式 | 示例 |
|------|------|
| `(数字)` | `photo (1).jpg`, `doc (2).pdf` |
| `[数字]` | `archive [1].zip` |
| `copy / copia / backup / kopia` | `report copy.docx` |
| ` - Copy` | Windows 英文系统复制后缀 |
| `副本 / 复件` | 中文系统复制后缀 |
| `_数字.` | `file_1.txt`, `data_2.csv` |

**路径扣分项**（命中 +2000 分）：

| 关键词 | 说明 |
|------|------|
| `backup`, `temp`, `tmp`, `cache` | 临时/备份目录 |
| `回收站`, `recycle`, `.trash`, `$recycle.bin` | 回收站/垃圾桶 |
| `.stversions` | Synology 版本历史 |

**基准分**：
- 文件名长度 × 2（优先短名）
- 路径长度 ÷ 10（轻微倾向短路径）

#### 3.4.2 信任目录白名单机制

用户可以指定一个或多个「信任目录」，这些目录中的文件被视为"原始文件"。

**规则**：
1. 每个重复组中，信任目录内的**所有**文件必须保留（无条件）
2. 信任目录外的所有文件一律标记为删除
3. 仅当组内没有任何文件位于信任目录时，才回退到策略选择保留 1 个

**示例**：
```
信任目录: C:\Photos\Original, D:\Archive

组内有:
  C:\Photos\Original\IMG001.jpg    ← 信任目录内 → 保留
  D:\Archive\IMG001.jpg            ← 信任目录内 → 保留
  E:\Temp\IMG001.jpg               ← 信任目录外 → 删除（唯一删除的）
```

**持久化**：信任目录列表保存到 `config.json`，启动时自动恢复。

## 5. 非功能需求

### 5.1 性能
- 扫描 10 万文件应在 3 分钟内完成（机械硬盘）
- 大文件（>1GB）不应阻塞 UI 线程
- 内存占用不超过 500MB（扫描 10 万文件场景）

### 5.2 安全
- 默认删除到回收站，不永久删除
- 系统文件/目录进入前发出警告
- 删除操作必须二次确认

### 5.3 兼容性
- 操作系统: Windows 10 / 11（64位）
- 支持中文路径和文件名
- 支持 NTFS、FAT32、exFAT 文件系统

### 5.4 可用性
- 界面语言: 中文
- 所有耗时操作需显示进度
- 错误信息需明确提示（无权限、路径不存在等）
